Tactical core details

METHODOLOGY & MATHEMATICS

1. Tactical Overview & Engineering Philosophy

The FIFA World Cup 2026 Tactical Fantasy Optimizer is a state-of-the-art recommendation engine designed to build and maintain an optimal fantasy squad. Instead of focusing on hyper-complex, overfit statistical predictions, our philosophy is anchored on a core principle:

Information breadth and news latency beat model complexity.

In short tournament formats, a player's actual starting likelihood—driven by injuries, coach press conferences, and suspension flags—carries far higher expected value (EV) than minor statistical variances. The engine is engineered to process massive volumes of real-time open web sources, extract structured availability intelligence, simulate the tournament, and run linear optimization algorithms to deliver clean, actionable recommendations.

2. Official Tournament Rules & Constraints

The engine models the official rules of the Aramco FIFA World Cup Fantasy 2026. Every recommendation respects the following strict constraints:

Squad Composition

Exactly 15 players: 2 Goalkeepers, 5 Defenders, 5 Midfielders, and 3 Forwards.

Budget Constraints

Initial budget of $100.0m. This increases automatically to $105.0m once Round 3 locks and the Round of 32 transfers open.

Country Cap

Maximum of 3 players per country during the group stage and Round of 32. This cap scales up in later rounds as countries are eliminated.

Transfer Penalties

2 free transfers are granted per matchday. Additional transfers can be made but incur a penalty of -3 points per transfer.

Tactical Formations

The starting XI must form a valid formation. The engine supports all official formations (e.g., 3-5-2, 4-4-2, 4-3-3, 3-4-3, 5-3-2, etc.).

Differential Scouting Bonus

Players with low ownership (< 5%) that score 4+ points in a match receive bonus points. The engine incorporates this differential boost into player evaluations.

3. Database & Data Architecture

The database of the Tactical Engine is built on a light, modular filesystem database designed for reliability, offline execution, and complete auditability:

  • Automated Data Pipelines: The engine queries the official FIFA API endpoints periodically to pull down players' prices, official positions, injury status, match fixture details, and deadlines.
  • Headlines Scraper: Google News RSS searches and major sports RSS feeds (such as BBC Sport) are scraped to compile articles and snippets about active squads.
  • Structured Event Database: Scraped articles are parsed (using keyword classifiers or LLMs) into structured events detailing player doubts, confirmed lineups, or suspensions.
  • State Storage (`data/state.json`): These events are blended with the official API data using a trust-weighted decay model to maintain the single source of truth for player states (likelihood to start, fitness index).
  • Audit Logging (`data/events.log`): Every single update to a player's starting probability is logged with timestamps, descriptions, and article URLs. No recommendation is made without receipts showing *why* it was suggested.

4. The Mathematical Prediction Engine

Expected points (xPts) and tournament survival calculations are solved using a combination of probability distributions, ratings-based goal expectations, and linear programming.

A. Team Goal Expectancy (λ)

Fixture strength is calculated using World Football Elo ratings. The Elo difference between teams is converted into a goal supremacy metric:

Diff = Elo_home - Elo_away + (Home_Advantage_Goals / 0.0035) Supremacy = Diff * 0.0035 λ_home = max(MIN_LAMBDA, (Average_Total_Goals / 2) + (Supremacy / 2)) λ_away = max(MIN_LAMBDA, (Average_Total_Goals / 2) - (Supremacy / 2))

Where Average_Total_Goals is set to a baseline of 2.60 goals per match, and MIN_LAMBDA = 0.20 goals.

B. Scoreline Joint Probability (Dixon-Coles Adjustment)

Standard Poisson models assume that the goals scored by each team are independent. However, football matches show a higher frequency of low-scoring draws ($0-0$ and $1-1$) than independent Poisson calculations suggest. To fix this, we apply the **Dixon-Coles adjustment**:

The joint probability of a scoreline $(h, a)$ is given by:

P(H = h, A = a) = τ(h, a) * Poisson(h; λ_home) * Poisson(a; λ_away)

Where the correlation adjustment factor τ(h, a) is defined using a correlation parameter ρ = -0.13:

τ(0, 0) = 1.0 - λ_home * λ_away * ρ τ(0, 1) = 1.0 + λ_home * ρ τ(1, 0) = 1.0 + λ_away * ρ τ(1, 1) = 1.0 - ρ τ(h, a) = 1.0 (for all other scorelines)

To prevent negative probabilities, $\tau$ is clipped at a minimum of $0.0$. This ensures that win, draw, and loss probabilities ($P_{win}, P_{draw}, P_{loss}$) are mathematically sound and align with real-world betting distributions.

C. Clean Sheet Marginals

An important mathematical property of the Dixon-Coles model is that it **preserves the marginal Poisson distributions**. The probability of a team conceding 0 goals remains:

P(Conceded = 0) = e^(-λ_against)

This keeps defensive clean sheet and conceded penalty calculations stable for expected fantasy points.

D. Tournament Survival (Monte Carlo Simulations)

Since fantasy managers need to avoid choosing players from countries that might be eliminated early, the engine runs a **10,000-trial Monte Carlo simulation** of the remaining tournament.

For each trial, the engine simulates group stage matches using Poisson goal draws, determines qualification standings, applies FIFA tiebreaker procedures, builds the Round of 32 bracket, and simulates knockout matches using Elo win probabilities. This yields:

P(Team Alive at Stage S) (for all S ∈ { R32, R16, QF, SF, F })

E. Player Expected Points (xPts)

For each unplayed fixture, a player's expected points is computed by blending their player priors (goal share, assist share, set-piece roles, and baseline stats per 90) with the team goal expectancies:

xPts(match) = P(start) * [ appearance_pts + goal_share * λ_for * goal_pts(pos) + assist_share * λ_for * assist_pts + P(Clean Sheet) * cs_pts(pos) + Conceded_Penalty_Term(pos) + stat_baseline(pos) + penalty_taker_flag * pen_goals_ev * goal_pts(pos) ] * fitness_mult + scouting_bonus_ev

F. Horizon Value & Mixed-Integer Linear Programming (MILP)

To make transfers, the optimizer does not look only at the next matchday; it maximizes the **Horizon Value (HV)** of the squad across the rest of the tournament:

HV = ∑ [ Discount^t * xPts_match(t) * P(Team Alive at Stage of Match t) ]

The squad selection, captaincy designation, and transfer swaps are solved as a Mixed-Integer Linear Program (MILP) using a branch-and-bound solver to find the mathematically optimal actions:

Maximize: ∑ HV(Player_i * Owned_i) - 3 * max(0, Transfers_Made - Free_Transfers) Subject to: - Budget ≤ $100.0m (or $105.0m) - Quotas: exactly 2 GK, 5 DEF, 5 MID, 3 FWD - Country Cap: ≤ stage limit per country - Valid formation constraints for Starting XI
